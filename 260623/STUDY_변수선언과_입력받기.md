## 1. 변수 선언 키워드 (let / const / var)

**변수** = 값을 담아두는 "상자". 상자를 만들 때 앞에 붙이는 키워드가 3개 있다.

| 키워드 | 뜻 | 재할당(값 바꾸기) | 실무 사용 |
| --- | --- | --- | --- |
| `const` | 안 바뀌는 상자 선언 | ❌ 불가능 | ✅ 기본으로 이걸 씀 |
| `let` | 바뀔 수 있는 상자 선언 | ✅ 가능 | ✅ 값이 바뀔 때만 |
| `var` | (옛날 방식) 바뀔 수 있는 상자 | ✅ 가능 | 🚫 요즘 안 씀 |

### 재할당이란?
한 번 넣은 값을 나중에 다른 값으로 바꾸는 것.

```js
let a = 1;
a = 2;        // ✅ 가능 (let은 바꿀 수 있음)

const b = 1;
b = 2;        // ❌ 에러! (const는 못 바꿈)
```

### let vs var 차이
- `var`는 옛날 거라 통하는 범위가 새어나가고, 같은 이름 재선언도 돼서 실수가 잦음 → 요즘 안 씀.
- `let`은 `{ }` 중괄호 안에서만 통해서 더 안전함.

### 실전 규칙
> 일단 `const`로 쓰고, 값을 바꿔야 할 일이 생기면 그때만 `let`으로. `var`는 옛날 코드에서만 보임.

## 2. 입력 받기 기본 틀 (Node.js)

```js
const fs = require("fs");
let input = fs.readFileSync(0).toString().trim().split('\n');
```

이 코드의 목적은 딱 하나: **"입력을 줄별로 잘라서 `input` 상자에 담는다."**
→ 외울 필요 없음. 문제 풀 때 맨 위에 복사해서 붙이는 '시작 주문' 같은 코드.

### 단어별 의미

| 단어 | 뜻 |
| --- | --- |
| `const fs = require("fs")` | 'fs'라는 입력 다루는 도구를 가져옴 (요리 전 칼 꺼내기) |
| `readFileSync(0)` | 입력을 통째로 읽음 (`0` = 키보드 입력) |
| `.toString()` | 읽은 걸 글자로 바꿈 |
| `.trim()` | 앞뒤 공백·엔터 잘라냄 |
| `.split('\n')` | 엔터(`\n`) 기준으로 잘라 줄별로 나눔 |
| `.` (점) | "그리고 그다음에" — 작업을 컨베이어처럼 이어붙임 |

### 값이 변해가는 과정 (입력이 `3` / `1 2 3` 일 때)

| 단계 | 코드 | 그 순간 값 |
| --- | --- | --- |
| 1 | `readFileSync(0)` | `"3\n1 2 3\n"` |
| 2 | `.toString()` | `"3\n1 2 3\n"` |
| 3 | `.trim()` | `"3\n1 2 3"` |
| 4 | `.split('\n')` | `["3", "1 2 3"]` |

결과 꺼내 쓰기 (칸 번호는 **0부터**):
```js
input[0]   // "3"
input[1]   // "1 2 3"
```

## 3. split = "기준 기호마다 가위로 자르기"

괄호 안에 적은 기호를 **가위 자국**으로 삼아 잘라서 배열로 만든다.

| 코드 | 자르는 기준 | 결과 |
| --- | --- | --- |
| `"3\n1 2 3".split('\n')` | 엔터 | `["3", "1 2 3"]` |
| `"1 2 3".split(' ')` | 빈칸 | `["1", "2", "3"]` |
| `"사과,배,귤".split(',')` | 쉼표 | `["사과", "배", "귤"]` |

> `\n` = 줄바꿈(엔터)을 글자로 표시한 것. 우리 눈엔 두 줄이지만 컴퓨터엔 `\n`이 낀 한 줄.

---

## 4. 실전 활용 예시

보통 첫 줄은 개수, 둘째 줄은 실제 숫자들인 경우가 많다.

```js
let n = Number(input[0]);        // "3" → 숫자 3
let arr = input[1].split(' ');   // "1 2 3" → ["1", "2", "3"]
```
