## ⭐ 핵심 발상 — "개념은 1개, 철자만 2개"

자바와 JS는 **하는 일은 같고 단어(철자)만 다름.** "하고 싶은 일"을 먼저 떠올리고 언어별 단어를 붙이면 안 헷갈린다.

| 하고 싶은 일 (개념) | 자바 (Java) | 자바스크립트 (JS) |
| --- | --- | --- |
| 변수 만들기 | `int a = 10;` | `let a = 10;` |
| 출력하기 | `System.out.println(a);` | `console.log(a);` |
| 입력 받기 | `Scanner sc = ...; sc.nextInt();` | `fs.readFileSync(0)...` |
| 글자→숫자 | `Integer.parseInt()` / `Double.parseDouble()` | `Number()` |
| 계산 | `a + b` (같음) | `a + b` (같음) |
| 비교 | `==`, `!=` | `===`, `!==` |

> 파일 확장자로 구분: `.java` = 자바 / `.js` = JS
> 가장 쉬운 구분: **`System.out.println` 보이면 자바, `console.log` 보이면 JS**

## 1. 값의 모양 (자료형) 4가지

같은 `15`라도 모양이 다르면 컴퓨터가 다르게 다룬다.

| 모양 | 표시 | 설명 |
| --- | --- | --- |
| 글자(String) | `"15"` | 따옴표로 감싼 것. 계산 안 됨 |
| 숫자(Number/int) | `15` | 따옴표 없음. 계산 됨 |
| 배열(Array) | `["3","1 2 3"]` | 대괄호. 칸칸이 담음 (0번부터) |
| 버퍼(Buffer) | `<Buffer 31 35>` | 컴퓨터 원본(암호). 사람이 못 읽음 |

함수는 전부 "모양 바꾸는 도구":
- `toString()` : 버퍼 → 글자
- `Number()` : 글자 → 숫자
- `split()` : 글자 → 배열

## 2. 글자 vs 숫자 (따옴표가 핵심)

- 따옴표 **있음** `"15"` → 글자 (계산 ❌)
- 따옴표 **없음** `15` → 숫자 (계산 ✅)

```js
15 + 2     // 17  (숫자라서 더해짐)
"15" + 2   // "152" (글자라서 옆에 붙음!)
```

> 그래서 입력받은 글자 `"15"`를 계산하려면 `Number("15")`로 숫자로 바꿔야 함.

## 3. `=`의 진짜 뜻 — "넣어라"

`=`는 "같다"가 아니라 **"오른쪽 값을 왼쪽 상자에 넣어라"** (대입).

```java
a = c   // "c값을 a에 넣어라"
```

### a=c vs c=a (방향 정반대!)
시작 a=1, c=3 일 때:

| 코드 | 뜻 | 결과 |
| --- | --- | --- |
| `a = c` | c → a | a=3, c=3 (a가 c로 덮어써짐) |
| `c = a` | a → c | a=1, c=1 (c가 a로 덮어써짐) |

> **왼쪽이 오른쪽 값으로 덮어써짐** (왼쪽 원래 값은 사라짐). 교환(swap)이 아니라 복사!

### 한국어 → 코드 번역
> "X의 값을 Y에 복사" = `Y = X` ("에" 붙은 게 왼쪽!)

### 빠르게 푸는 법
1. `=` 보이면 → `←` "넣어라"로 읽기
2. 지문은 "에" 붙은 게 왼쪽
3. **코드 옆에 값 적으며** 한 줄씩 (머리로만 X)

## 4. char vs String (자바)

자바는 따옴표 2종류를 다르게 봄:

| 따옴표 | 이름 | 뜻 |
| --- | --- | --- |
| `' '` 작은따옴표 | char | 딱 한 글자 (`'C'`) |
| `" "` 큰따옴표 | String | 글자 묶음 (`"CBA"`) |

```java
char c = 'C';      // ✅ 한 글자
char c = "C";      // ❌ String을 char에 (모양 안 맞음)
String c = "CBA";  // ✅ 여러 글자
String c = CBA;    // ❌ 따옴표 없으면 변수로 오해
```

> 참고: `int c = 'C';` 는 됨! 단 글자가 아니라 번호 `67`로 저장됨 (컴퓨터는 글자를 번호로 가짐 = 아스키코드).
> 누가 정함? Java 만든 사람들(언어 명세). 왜? char/String은 저장 방식이 달라서 구분.

## 5. 출력 — print / println / printf (자바)

| 함수 | 뜻 | 빈칸(%d) 채우기 | 줄바꿈 |
| --- | --- | :---: | :---: |
| `print` | 출력만 | ❌ | ❌ |
| `println` | print + line | ❌ | ✅ |
| `printf` | print + format | ✅ | ❌ (`\n`으로) |

### printf 빈칸 종류 (영어 첫 글자!)
| 빈칸 | 영어 | 들어갈 것 |
| --- | --- | --- |
| `%d` | decimal | 정수 |
| `%f` | float | 소수 |
| `%s` | string | 글자 |
| `%c` | char | 한 글자 |

> printf는 `%` 개수 = 값 개수 맞춰야 함. `printf("%d", 10)` 처럼 채울 값 같이 주기.
> println은 값 1개만 받음 → 여러 개는 `+`로 이어붙이기: `println(a + " " + b)`

## 6. 입력 받기 (Scanner - 자바)

```java
import java.util.Scanner;                  // ① 한 번만 (맨 위)
Scanner sc = new Scanner(System.in);       // ② 한 번만 (도구 만들기)
int a = sc.nextInt();                      // ③ 입력받을 때마다 여러 번
```

왼쪽 타입 ↔ 오른쪽 메서드 **짝 맞추기**:

| 받을 것 | 왼쪽 타입 | 오른쪽 메서드 |
| --- | --- | --- |
| 정수 | `int` | `sc.nextInt()` |
| 소수 | `double` | `sc.nextDouble()` |
| 참/거짓 | `boolean` | `sc.nextBoolean()` |
| 단어 | `String` | `sc.next()` |
| 한 줄 | `String` | `sc.nextLine()` |

> Scanner 쓰면 toString/parseInt 같은 변환 필요 없음 (알아서 숫자로 줌).

## 7. 산술 연산자 (계산 기호)

a=7, b=2 기준:

| 기호 | 뜻 | 결과 |
| --- | --- | --- |
| `+` | 더하기 | 9 |
| `-` | 빼기 | 5 |
| `*` | 곱하기 (×아님) | 14 |
| `/` | 나누기(몫) | 3 |
| `%` | 나머지 | 1 |

> ⚠️ 자바에서 **정수끼리 `/`는 몫만** (7/2=3, 3.5 아님). 나머지는 `%`로.
> 우선순위: `* / %` 먼저 → `+ -` 나중. 순서 바꾸려면 괄호 `( )`.
> `%`(나머지) 활용: 짝/홀 판단 → `n % 2 == 0` 이면 짝수.

## 8. 스코프 (변수가 통하는 범위)

`{ }` = 방. 방이 다르면 같은 이름 변수 또 써도 OK.

```js
let a = "이런";          // 바깥 방
function test() {
    let a = "저런";      // 함수 안 방 (다른 방이라 OK)
}
```

| | 다른 방 같은 이름 | `{ }` 방 지킴? | 추천 |
| --- | :---: | :---: | :---: |
| `var` | ✅ | ❌ 무시(위험) | 🚫 옛날 거 |
| `let` | ✅ | ✅ 지킴 | ✅ 값 바뀔 때 |
| `const` | ✅ | ✅ 지킴 | ✅ 기본 |

규칙:
- **같은 방에서 `let` 두 번 = 에러** (한 방에 같은 이름 하나만)
- 같은 방에서 값만 바꿀 땐 `let` 빼고 이름만: `a = "저런";` (재할당)
- **`const`는 만들 때 값 필수** (`let`은 비워도 됨)
- 함수는 `function 이름() {}` (이름 꼭!)

## 9. 글자 → 숫자 변환

| 하고 싶은 일 | JS | 자바 |
| --- | --- | --- |
| 글자 → 정수 | `Number()` | `Integer.parseInt()` |
| 글자 → 소수 | `Number()` | `Double.parseDouble()` |

- JS는 숫자가 한 종류(number)라 `Number()` 하나로 정수·실수 다 됨.
- 자바는 int/double 따로라 변환도 둘로 나뉨.
- `parseInt` = parse(해석) + Int(정수). 정수만 만듦 (`"3.14"` → `3`, 소수점 버림).

## 10. 비교 연산자 (JS) — `!=` vs `!==`

| 기호 | 비교 |
| --- | --- |
| `!=` / `==` | 값만 (모양 무시) |
| `!==` / `===` | 값 + 모양(타입) |

```js
5 != "5"    // false (값만 보면 같음)
5 !== "5"   // true  (숫자 vs 글자, 모양 다름)
```

> 추천: **`===`, `!==` (= 3개, 엄격한 것)** 쓰기. `==`는 예상 못한 결과 남 (`0 == ""` → true).

## 11. 함수 이름 = 영어 뜻

| 함수 | 영어 뜻 | 하는 일 |
| --- | --- | --- |
| `read` | 읽다 | 읽어오기 |
| `toString` | to(~로)+String(글자) | 글자로 바꾸기 |
| `trim` | 다듬다 | 앞뒤 공백·엔터 자르기 |
| `split` | 쪼개다 | 기준대로 자르기 |
| `parse` | 해석하다 | (해석해서) 바꾸기 |

> `split('\n')` = 엔터 기준 / `split(' ')` = 빈칸 기준 / `split('')` = 글자 하나하나 / `\n` 없으면 통째로 1칸 배열.
